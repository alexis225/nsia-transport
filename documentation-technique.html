Principe multi-tenant

Chaque filiale est un tenant identifié par un UUID (tenant_id). L'isolation est assurée à deux niveaux :

Niveau applicatif

Middleware EnsureTenantIsolation + scopes Eloquent filtrant automatiquement les requêtes par tenant_id.

Niveau base de données (RLS)

Row Level Security PostgreSQL activé sur les tables critiques. SetTenantContext injecte app.current_tenant_id en session PostgreSQL à chaque requête.

Architecture de SécuritéDéfense en profondeur — US-050 à US-052

1. Couche réseau

  • Middleware CheckIpBlacklist (global web)
  • Cache 5 min par IP (performance)
  • Support CIDR PostgreSQL ::inet <<=
  • Verrouillage compte : 5 échecs → 30 min

2. Couche authentification

  • Fortify : Bcrypt + rate limiting
  • MFA TOTP (Fortify natif, chiffré)
  • SSO Microsoft (Azure AD single-tenant)
  • Sessions PHP + CSRF token

3. Couche autorisation

  • Spatie Permission teams=true
  • Gate::before() : super_admin bypass
  • Délégation via UserRoleGrant
  • Middleware permission: sur routes

4. Couche données

  • RLS PostgreSQL (tables critiques)
  • AsEncryptedString sur champs PII
  • UUID v4 pour toutes les PKs
  • Soft delete (audit trail)

Flux d'autorisation

// AppServiceProvider::boot()
Gate::before(function ($user, $ability) {
    // Super admin bypass TOTAL
    if ($user->hasRole('super_admin')) return true;

    // Délégation temporaire de rôle
    if (UserRoleGrant::hasGrantedPermission($user, $ability)) return true;

    return null; // laisse Spatie Permission décider
});

Chiffrement des données sensibles (PII)

// Sur les modèles Broker et InsuranceContract
protected function casts(): array
{
    return [
        'address'             => AsEncryptedString::class, // chiffré en base
        'phone'               => AsEncryptedString::class,
        'registration_number' => AsEncryptedString::class,
    ];
}
// ⚠️ email = NON chiffré (utilisé pour login/recherche)

Services, Jobs & ObserversLogique métier découplée

Services

CommissionService

Calcule la commission courtier à l'émission d'un certificat. Priorité : règle contrat > règle courtier. Évite les doublons. Annule si certificat annulé.

Déclenche CommissionService::calculate() automatiquement quand le statut passe à ISSUED. Enregistré dans AppServiceProvider.

DéploiementChecklist de mise en production

1. Préparer l'environnement

PHP 8.2+, PostgreSQL 16, Node 20+, Composer, queue worker (Supervisor)

2. Variables d'environnement

Copier .env.example → .env. Configurer DB, Mail, Microsoft OAuth, APP_KEY.

3. Dépendances

composer install --no-dev --optimize-autoloader
npm ci && npm run build

4. Base de données

php artisan migrate --force
php artisan db:seed --class=RolesAndPermissionsSeeder

5. Chiffrement PII (1 fois)

php artisan nsia:encrypt-sensitive-fields — à exécuter une seule fois après migration initiale des données.

6. Optimisations Laravel

php artisan config:cache && php artisan route:cache && php artisan view:cache

7. Stockage

php artisan storage:link — lien symbolique public/storage → storage/app/public

8. Scheduler (crontab)

* * * * * php /path/to/artisan schedule:run >> /dev/null 2>&1

9. Queue worker (Supervisor)

Configurer Supervisor pour php artisan queue:work --sleep=3 --tries=3

10. Vérification finale

Tester login, MFA, émission certificat, PDF, export. Vérifier RLS PostgreSQL actif.

Checklist RLS : Vérifier que la migration 2026_06_01_000001_enable_row_level_security.php est bien exécutée et que les politiques PostgreSQL sont actives sur les 5 tables couvertes.
